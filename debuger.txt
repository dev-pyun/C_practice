디버거 : 버그를 제거하는 도구

디버거를 사용한다면 C언어를 훨~씬 쉽게 배울 수 있음.

중단점이란?
소스코드의 특정 지점에서 프로그램의 실행을 멈추는데 사용.

사용전, 솔루션을 확인할 것.
vs의 윗부분에 Debug, X86으로 설정되어있는지 확인하기.

중단점 만드는 3가지 방법
1. 원하는 줄에서 마우스 우클릭 - 중단점 - 중단점 설정
2. 코드 옆 회색 부분에서 실행을 멈추기를 원하는 곳을 클릭.
3. 원하는 줄에서 F9를 눌러 중단점 설정하기.

이후 F5키를 눌러 디버깅 시작하기.
그러다가 중단점을 설정한 부분에서 코드실행이 일시정지됨.
이때 F10키를 누르면 한줄씩 코드를 실행할 수 있음.

정리 :
중단점 삽입/삭제 : F9
디버깅 시작 : F5
디버깅 중지 : Shift + F5
프로시저 단위 실행 : F10
한단계씩 코드 실행 : F11 (여기서 단계는 주로 함수를 뜻함.)

Dev-C++에서도 디버거를 사용할 수 있으며 단축키는 다음과 같습니다(버전 5.11 이상 기준).


중단점 활성/비활성(Visual Studio 중단점 삽입/삭제에 해당): F4
디버그(Visual Studio의 디버깅 시작에 해당): F5
디버깅 중단(Visual Studio의 디버깅 중지에 해당): F6
다음 단계(Visual Studio의 프로시저 단위 실행에 해당): F7
단계 정보(Visual Studio의 한 단계씩 코드 실행에 해당): F8
